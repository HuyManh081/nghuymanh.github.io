Phân biệt JVM, JRE và JDK dễ hiểu nhất

Huy Mạnh | Java Core | 03/06/2024

Giới thiệu: Khi mới học Java, bạn sẽ bị "xoay như chong chóng" bởi các thuật ngữ viết tắt: JVM, JRE, JDK. Chúng khác nhau thế nào? Tại sao tôi tải JDK mà trong đó lại có JRE? Bài viết này sẽ giúp bạn làm sáng tỏ mọi thứ.

1. Tổng quan mối quan hệ

Hãy tưởng tượng mối quan hệ bao hàm như những chiếc hộp lồng vào nhau:

JDK
(Chứa JRE + Công cụ phát triển)
JRE
(Chứa JVM + Thư viện chuẩn)
JVM
(Máy ảo Java)

2. JVM (Java Virtual Machine) - Máy ảo Java

Đây là "trái tim" của Java. JVM là một chương trình phần mềm mô phỏng một máy tính thật.

  • Nhiệm vụ: Nó không hiểu code Java bạn viết (file .java). Nó chỉ hiểu Bytecode (file .class). Nhiệm vụ của nó là dịch Bytecode này thành mã máy (Machine Code) để hệ điều hành (Windows/Mac/Linux) có thể chạy được.
  • Tại sao cần JVM? Nhờ JVM mà Java có tính năng "Write Once, Run Anywhere". Bạn viết code trên Windows, biên dịch ra Bytecode. Mang file Bytecode đó sang Mac hay Linux, chỉ cần máy đó có JVM cài sẵn là chạy được ngay.

3. JRE (Java Runtime Environment) - Môi trường chạy

JRE là bộ phần mềm cần thiết để CHẠY một ứng dụng Java (nhưng không dùng để lập trình).

  • Thành phần: JRE = JVM + Các thư viện thư viện chuẩn (Java Class Libraries) + Các file cấu hình.
  • Đối tượng sử dụng: Người dùng cuối (End-User). Ví dụ: Bạn muốn chơi game Minecraft (viết bằng Java), bạn chỉ cần cài JRE là đủ.

4. JDK (Java Development Kit) - Bộ công cụ phát triển

JDK là bộ công cụ đầy đủ dành cho Lập trình viên.

  • Thành phần: JDK = JRE + Các công cụ phát triển (Compiler javac, Debugger, Javadoc...).
  • Đối tượng sử dụng: Developer. Nếu bạn muốn viết code và biên dịch nó, bạn bắt buộc phải cài JDK.

5. Tóm lại

Thuật ngữ Tên đầy đủ Vai trò chính
JVM Java Virtual Machine Dịch Bytecode thành mã máy để chạy trên từng OS cụ thể.
JRE Java Runtime Environment Môi trường để chạy ứng dụng (Dành cho User).
JDK Java Development Kit Bộ đồ nghề để viết và biên dịch code (Dành cho Dev).
