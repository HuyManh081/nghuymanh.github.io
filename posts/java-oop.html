4 Trụ cột của Lập trình hướng đối tượng (OOP)

Huy Mạnh | Java Core | 02/06/2024

Giới thiệu: Java là ngôn ngữ thuần hướng đối tượng. Để code Java tốt, bạn không chỉ cần nhớ cú pháp mà phải tư duy theo OOP. Mọi thứ xoay quanh 4 tính chất: Đóng gói, Kế thừa, Đa hình và Trừu tượng.

1. Tính Đóng gói (Encapsulation)

Là việc che giấu dữ liệu bên trong object và chỉ cho phép truy cập thông qua các phương thức công khai (Getter/Setter).

  • Mục đích: Bảo vệ dữ liệu, không cho bên ngoài sửa đổi tùy tiện.
  • Cách dùng: Đặt thuộc tính là private, phương thức là public.
public class BankAccount {
    private double balance; // Người ngoài không thể sửa trực tiếp

    public void deposit(double amount) {
        if (amount > 0) balance += amount; // Kiểm soát logic nạp tiền
    }
}

2. Tính Kế thừa (Inheritance)

Cho phép class con tận dụng lại (kế thừa) các thuộc tính và phương thức của class cha. Giúp giảm lặp code (DRY - Don't Repeat Yourself).

  • Từ khóa: extends.
  • Lưu ý: Java chỉ hỗ trợ đơn kế thừa (một con chỉ có một cha).

3. Tính Đa hình (Polymorphism)

Một hành động có thể được thực hiện theo nhiều cách khác nhau. Đa hình thể hiện qua 2 cơ chế:

  • Overloading (Nạp chồng): Cùng tên hàm nhưng khác tham số (Trong cùng 1 class).
    Ví dụ: tinhTong(int a, int b) và tinhTong(double a, double b).
  • Overriding (Ghi đè): Class con viết lại hàm của class cha (Khác class).
    Ví dụ: Cha là DongVat có hàm keu(), con là Cho ghi đè lại thành "Gâu gâu", Meo ghi đè thành "Meo meo".

4. Tính Trừu tượng (Abstraction)

Là việc ẩn đi các chi tiết cài đặt phức tạp, chỉ hiển thị những tính năng cần thiết cho người dùng. Nó tập trung vào câu hỏi: "Đối tượng này làm được gì?" thay vì "Nó làm thế nào?".

Trong Java, trừu tượng thể hiện qua:

  • Abstract Class: Class có thể chứa hàm trừu tượng (không có thân hàm) và hàm thường.
  • Interface: Bản thiết kế (blueprint) chỉ chứa các phương thức rỗng, buộc các class khác phải implements (triển khai) nó.
